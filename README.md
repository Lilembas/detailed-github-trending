1. **项目功能**

2. **关键特性**  
- ✅ **精准溯源定位**：为每个提取结果标注在源文本中的起始/结束字符偏移，支持可视化高亮与人工核查；

- ✅ **长文档优化处理**：采用智能分块（chunking）、多轮提取（up to 3+ passes）、并行处理（`max_workers` 可调）策略，显著提升超长文本（如整本小说、完整医学报告）的召回率与准确性；  
- ✅ **交互式可视化审查**：一键生成自包含、可离线运行的 HTML 文件，支持动态搜索、上下文展开、实体聚类与动画高亮，便于大规模结果人工复核；  
- ✅ **多模态 LLM 接入能力**：原生支持 Google Gemini（云）、OpenAI（需额外安装）、本地 Ollama 模型（如 `gemma2:2b`），并提供插件化架构供用户轻松扩展第三方模型提供商；  
- ✅ **零微调领域适配**：仅需提供 1–3 个高质量、按出现顺序排列的文本-提取对示例，即可快速适配任意垂直领域（医疗、法律、文学分析等），无需训练或微调模型；

Claude-Mem 是一个专为 Anthropic 的 **Claude Code**（AI 编程助手）构建的**持久化记忆压缩系统**。它在用户每次会话结束后，自动捕获工具调用、代码操作、错误修复等上下文事件（称为“observations”），生成语义化摘要，并持久化存储于本地 SQLite 数据库中；当新会话启动时，智能检索并按需注入相关历史记忆，从而实现跨会话的**上下文连续性**——使 Claude 能“记住”项目进展、技术决策、调试过程等关键知识，显著提升长周期开发任务中的推理连贯性与效率。

2. **核心特性**  
- ✅ **持久记忆（Persistent Memory）**：会话结束后仍保留结构化记忆，重启即恢复上下文；  
- ✅ **渐进式披露（Progressive Disclosure）**：分三层（索引 → 时间线 → 原始详情）按需加载记忆，实时显示 token 消耗，优化成本；  
- ✅ **技能化搜索（mem-search 技能）**：支持自然语言查询历史（如 `search("authentication bug", type="bugfix")`），结合时间、类型、项目等多维过滤；  
- ✅ **Web 可视化界面（http://localhost:37777）**：实时流式展示记忆流、观测记录、摘要及全文，支持 ID 引用与 API 直查；  
- ✅ **Claude Desktop 集成**：可在桌面版对话中直接调用 `mem-search` 技能检索记忆；  
- ✅ **隐私保护机制**：通过 `<private>` 标签自动排除敏感内容，不存入数据库；  
- ✅ **全自动运行**：无须手动触发，依赖 6 个生命周期钩子（SessionStart/PostToolUse/SessionEnd 等）静默采集；  
- ✅ **混合搜索架构**：融合 SQLite FTS5 全文检索 + Chroma 向量数据库，兼顾关键词精准性与语义相关性；
